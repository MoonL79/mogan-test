# 开发规范

## 技术栈

- 本仓库默认使用 **Goldfish Scheme** 实现。
- 对应命令行工具是 `gf`。
- 不应默认引入 Python、Node.js、Go 或其他语言实现主平台逻辑，除非 `spec.md` 明确允许。

## 工作流

1. 在开始具体任务前，先读取本仓库根目录的 `CLAUDE.md` 与 `spec.md`。
2. 所有实现都应优先保持最小可用范围。
3. 先定义稳定接口，再补内部实现。
4. 先保证可测试、可路由、可扩展，再考虑补充功能。

## Goldfish Scheme 约束

- 优先使用 `gf doc` 和 `gf source` 查询库与源码。
- 代码使用 2 空格缩进，不使用 Tab。
- 所有跨行环境必须带右侧标记。
- 保持结构扁平，避免不必要的深层嵌套。
- 命名遵循 `kebab-case`。
- 谓词函数使用 `?` 后缀。
- 副作用函数使用 `!` 后缀。

## 测试规范

- 优先提供可脚本化、可重复执行的测试入口。
- 新能力应尽可能提供最小验证用例。
- 若当前阶段只能搭建骨架，必须明确哪些部分是 stub，哪些部分可验证。
